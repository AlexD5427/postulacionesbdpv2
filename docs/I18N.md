# Internacionalización (i18n) — 4 idiomas

El portal se ofrece en **cuatro idiomas** en toda la interfaz visible por el usuario:

| Código | Idioma | Endónimo |
| --- | --- | --- |
| `es` | Español (Latinoamérica) — **fuente de verdad** | Español |
| `en` | Inglés | English |
| `qu` | Quechua (Runa Simi, sur de Bolivia) | Runa Simi |
| `ay` | Aymara | Aymar Aru |

## Regla no negociable

> 🔒 **Toda cadena de texto nueva que se vea en el frontend a nivel de usuario DEBE**
> añadirse a `es` (la fuente de verdad) y traducirse a **inglés, quechua y aymara**.
> No se aceptan literales de texto sueltos en los componentes: todo pasa por `t('clave')`.

Las claves ausentes en `qu`/`ay` (o `en`) **caen con elegancia al español** mediante la cadena
de respaldo `locale → es → clave`. Ese respaldo es una red de seguridad, **no** una licencia para
omitir la traducción. La cobertura de quechua y aymara del MVP cubre el núcleo de alta visibilidad
(marca, preloader, navegación, dock, héroe, CTAs, centro de accesibilidad, tema); el resto usa el
respaldo y debe completarse. **Se recomienda revisión de hablantes nativos** antes de producción.

## Arquitectura

```
src/features/i18n/
├── config.ts              # LOCALES, DEFAULT_LOCALE, LOCALE_META
├── storage-key.ts         # 'bdp.locale.v1' (módulo server-safe, sin 'use client')
├── dictionaries/
│   ├── es.ts              # fuente de verdad → define TranslationKey y Dictionary
│   ├── en.ts / qu.ts / ay.ts  # Partial<Dictionary>
├── dictionary.ts          # translate(locale, key, vars) + cadena de respaldo
├── i18n-store.ts          # Zustand persist + applyLocaleToDocument (<html lang>)
├── use-translation.ts     # hook useTranslation() → { t, locale, setLocale }
└── LanguageSwitcher.tsx   # menú (navbar) e inline (panel de accesibilidad)
```

### Seguridad de tipos

`es` se declara `as const`; a partir de él se derivan:

```ts
export type TranslationKey = keyof typeof es;      // unión de todas las claves
export type Dictionary = Record<TranslationKey, string>;
```

Los demás diccionarios son `Partial<Dictionary>`, así que **TypeScript marca las claves mal
escritas** y permite cobertura parcial. Interpolación con `{var}`:
`t('saludo', { name: 'Ana' })`.

### Seguridad de hidratación

El idioma vive en `localStorage`, que el servidor no puede leer. Para evitar un desajuste de
hidratación, el **primer render** (servidor y primer render de cliente) usa siempre el español; tras
`useEffect` se aplica el idioma persistido. Un script en línea en `layout.tsx` fija `<html lang>`
antes del primer pintado para lectores de pantalla.

## Cómo añadir texto nuevo (checklist)

1. Añade la clave a `dictionaries/es.ts` (agrupada y con espacio de nombres por punto).
2. Añade su traducción a `en.ts`, `qu.ts` y `ay.ts`.
3. Úsala con `const { t } = useTranslation();` → `t('mi.clave')`.
4. Si el componente es de servidor, envuélvelo en un componente de cliente o pásale el texto ya
   traducido desde un componente de cliente.

## Cambiar de idioma

`LanguageSwitcher` (globo 🌐 en la barra y control segmentado en el centro de accesibilidad)
escribe en el store; `applyLocaleToDocument` actualiza `<html lang>` en caliente.
